LV.RegisterModule

Overview:

 

The LV.RegisterModule() command is used to load a MLua module and register it with the current MLua instance. Similar to the Register Module VI.

 

Usage:

 

For convenience the command comes in three variations, the number of arguments determine which version is invoked.

 

1.Complete call with three arguments:

 

LV.RegisterModule("path_to_module", "module_name", user_data)

 

With three arguments the module that is found at "path_to_module" is registered with the name "module_name", the Lua data provided in user_data is used as the custom user data for the module.

 

example:

 

 -- create path to MLua module

path = 'C:\test module.vi'

 -- load and register module

LV.RegisterModule(path, 'test', 123.456)

 

Registering a MLua module from disk

 

 

2. Call with two arguments:

 

LV.RegisterModule("path_to_module", user_data)

 

With two arguments the module that is found at "path_to_module" is registered either as the new default module if it has no "__name" property or under the returned string as its name, the Lua data provided in user_data is used as the custom user data for the module.

 

example:

 

 -- create path to MLua module

path = 'C:\test module.vi'

 -- load and register module

LV.RegisterModule(path, 123.456)

 

3. Call with a single argument

 

LV.RegisterModule("path_to_module")

 

With one argument the module that is found at "path_to_module" is registered either as the new default module if it has no "__name" property or under the returned string as its name.

 

example:

 

 -- create path to MLua module

path = 'C:\test module.vi'

 -- load and register module

LV.RegisterModule(path)
