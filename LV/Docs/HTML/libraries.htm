Libraries

Overview:

 

MLua allows the use of external Lua libraries and compiled C modules. This greatly increases code reuse and also provides a flexible counterpart to LabVIEW's Call Library Function Node. For more information regarding Lua libraries visit: Lua 5.2

 

Example 1:

 

The example below shows how Lua modules are loaded. It is important to realize that by default Lua searches for modules relative to the current executable. In the case of running this example in the LabVIEW development environment, Lua will begin searching for modules where LabVIEW.exe is located. MLua provides functions to easily change the search patterns to suite different applications. Refer to Get Package Paths and Set Package Paths.
