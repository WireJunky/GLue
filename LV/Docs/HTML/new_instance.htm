New Instance




Creates a new instance of MLua which can be wired to other MLua API functions.

Parameter(s)
-----------------

- Pool size: Amount of memory to allocate to the MLua instance, Defaults to '0'. A value of '0' disables the fixed memory function allowing Lua to dynamically allocate memory as required.

- error in: Describes error conditions that occur before this node runs. This input provides standard error in functionality.

- MLua out: Newly created MLua instance.

- error out: Contains error information. This output provides standard error out functionality.

 

Overview:

 

The New Instance VI creates a new MLua instance, which is isolated from other MLua instances. The other VIs that make up the MLua API requires a valid instance created using the New Instance VI. In addition, a fixed amount of memory can be provided for the instance by wiring a value to the pool size (bytes) input terminal. A value of '0' disables the fixed memory facility and allows the script to dynamically allocate memory as needed.

 

Usage:

 

Creating a new MLua instance

 

In the example above, a new instance is created with a pool size of '0'. By wiring the instance to two parallel Execute Script functions, the operations are serialized. if parallel execution of scripts is required then separate MLua instances must be created (as seen below).
