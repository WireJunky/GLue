LV.Cluster

Overview:

 

The LV.Cluster() command is used to create a Lua table which is to be converted into a LabVIEW cluster. This conversion happens when the LV.Cluster is passed back as a return from a script or when a LV.Call is performed with the LV.Cluster as part of the argument data. The LV.Cluster command adds a cluster type tag to the resulting table, it also adds meta methods to transparently record the order of added elements. This is needed as Lua tables (that are not arrays) do not guarantee item order after insertion.

 

The fact that Lua tables do not guarantee item order also makes it difficult to simply pass Lua tables to LabVIEW to convert into a cluster. If it is a plain Lua table without the LV.Cluster meta information, then it will still be converted into a cluster, but the order of the elements will not always be the same.

 

Usage:

 

The command creates a new empty Lua table with the required meta information indicating it is to be treated as a LabVIEW cluster. LV.Clusters can accept any Lua data type, including other LV types such as LV.Variant and LV.StringArray.

 

1.Create cluster:

 

 -- create a new cluster table

lcluster = LV.Cluster()

lcluster.string = 'hallo world'

 -- create a child cluster

lchild = LV.Cluster()

 -- add number to child cluster

lchild.var = 123.456

 -- insert child cluster

lcluster.child = lchild

return lcluster
