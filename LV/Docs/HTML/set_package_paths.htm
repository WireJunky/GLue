Set Package Paths




Set the paths to search for Lua and C modules, see http://www.lua.org/manual/5.2/manual.html for further details.

Parameter(s)
-----------------

- MLua in: Reference to the MLua instance.

- Lpaths: A semi-colon seperated list of possible paths to Lua script modules.

- Cpaths: A semi-colon seperated list of possible paths to compiled C modules.

- error in: Describes error conditions that occur before this VI runs. This input provides standard error in functionality.

- MLua out: Reference to the MLua instance.

- error out: Contains error information. This output provides standard error out functionality.

 

Overview:

 

Lua uses search patterns to look for Lua library files and compiled C modules. This happens when "require" is called from within a script. MLua allows these search paths to be updated at run time, the format of which is explained in detail at http://www.lua.org/manual/5.2/manual.html#6.3.

 

Search path example: "./?.lua;./?.lc;/usr/local/?/init.lua"

 

In the example above the search path for lua library files is shown. It is semi-colon separated list, where the ? is replaced with the value supplied to the require statement in the script. By issuing the command require "test" in the script, the value test is substituted into the ? placeholder. Therefore if the Lua loader finds the file at <path to executable>\test.lua then it stops and loads the file, if not found it continues searching using the next item in the search list. The same applies to the search patterns for compiled C modules (.dll).

 

Usage:

 

Setting the paths used to search for dependencies
