LV.Variant

Overview:

 

The LV.Variant() command is used to create a Lua table which is to be converted into a LabVIEW variant. This conversion happens when the LV.Variant is passed back as a return from a script or when a LV.Call is performed with the LV.Variant is part of the argument data. The LV.Variant command adds a variant type tag to the resulting table, which allows MLua to correctly identify and convert the value into a LabVIEW variant. The result of the LV.Variant command is a table, but only the first valid entry in the table will be converted into a LabVIEW variant.

 

Usage:

 

The command takes a single argument and returns a Lua table with the required meta information indicating it is to be treated as a LabVIEW variant. LV.Variants are designed to increase LV.Cluster flexibility. When the Lua cluster table is converted into a LabVIEW cluster the LV.Variant's data is converted into valid LabVIEW data, but this data is inserted into the LabVIEW cluster wrapped as a variant. LV.Variants can accept any Lua data type, including other LV types such as LV.Cluster and LV.StringArray.

 

1.Create variant:

 

 -- create a new cluster table as described in LV.Cluster

lcluster = LV.Cluster()

lcluster.string = 'hallo world'

 -- create a new variant that wraps a numeric constant

lvariant = LV.Variant(123.456)

 -- add variant to cluster

lcluster.var = lvaraint

return lcluster
